{vars.company.short} offers a public API for access to all the services available through the {vars.portal.long}. You can use this API to automate your {vars.company.short} provisioning to save time and effort.

The {vars.company.short} API is based on REST. The API has predictable URLs, accepts form-encoded request bodies, returns json-encoded responses, and uses standard HTTP response codes.

The topics in this section step you through the API calls needed to perform common tasks such as logging in, creating a Port, and creating a VXC from the Port to a cloud provider.
